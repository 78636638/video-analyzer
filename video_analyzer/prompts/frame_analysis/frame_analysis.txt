Output Rules
- 先输出一个紧凑、合法的 JSON 对象。
- 再在 `Narrative:` 下输出简短说明。
- JSON 键名保持英文，字段值与 Narrative 必须使用 `{OUTPUT_LANGUAGE}`。
- 如果同一人物已经在近期记忆中出现，优先复用稳定称谓或角色描述。
- 不要编造具体人名；如果近期记忆或附近音频里没有已验证名字，`name` 留空，改用 `role` 或外观特征。

JSON Schema
{
  "scene": "short scene label",
  "characters": [
    {
      "name": "character name or empty string",
      "appearance": ["visible appearance clues"],
      "role": "role or relation if known"
    }
  ],
  "actions": ["key visible actions"],
  "objects": ["important objects or props"],
  "dialogue_hint": "visible or audio-supported dialogue clue",
  "continuity_points": ["what should be tracked next"],
  "scene_changed": false
}

Writing Guidance
- 使用现在时。
- 保持客观、克制。
- 聚焦可见动作、场景变化、人物连续性和关键道具。
- 如果当前帧与最近窗口高度相似，强调“延续”而不是重复描述。
- 如果人物身份不确定，使用“哥哥、工人、人物”等通用称谓，避免生成新的专有姓名。
